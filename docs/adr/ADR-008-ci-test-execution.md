# ADR-008: GitHub Actions での自動テスト実行

## ステータス

Accepted

## コンテキスト

現在の GitHub Actions ワークフロー（`build.yml`）は Xcode ビルドチェックのみを実行しており、
テストは CI で実行されていない。ADR-006/007 で整備した 87 件のユニットテストを CI で
自動実行し、品質ゲートとして機能させる必要がある。

### 検討事項

1. **テスト実行時間**: `swift test` で全 87 テストが **0.435 秒**で完了（ビルド含めても数秒）
2. **テスト分離の必要性**: テスト総時間が 1 秒未満と極めて短いため、必須/追加テストの分離は不要
3. **実行環境**: テストは `Package.swift` + `#if canImport()` ガードにより Linux 上で動作可能
4. **既存ビルドジョブ**: macOS 14 上の Xcode ビルドチェックは iOS 固有の検証として維持

## 決定

### 全テストを CI で実行する

テスト総時間が 0.435 秒と極めて短いため、テストの分離は行わず **全テストを毎回実行** する。

### Ubuntu ランナーでの `swift test` ジョブ追加

既存の `build.yml` に `test` ジョブを追加する。

- **ランナー**: `ubuntu-latest`（macOS ランナーより起動が高速・コスト効率が良い）
- **Swift バージョン**: `swift:5.9` Docker イメージを使用
- **実行コマンド**: `swift test`
- **並列実行**: `build` ジョブと `test` ジョブは独立して並列実行

```yaml
test:
  name: Swift Test (Linux)
  runs-on: ubuntu-latest
  container: swift:5.9
  steps:
    - uses: actions/checkout@v4
    - run: swift test
```

### ワークフロー名の変更

テストが追加されるため、ワークフロー名を `Build Check` から `CI` に変更する。

## 代替案

### 却下: テストの必須/追加分離

テスト総時間が 1 秒未満のため、分離によるオーバーヘッド（設定の複雑化）が
メリットを上回る。将来テストが増加して実行時間が問題になった場合に再検討する。

### 却下: macOS ランナーでの swift test

macOS ランナーは起動時間が長く（30秒〜1分）、コストも高い。
テストは Linux 互換に設計されているため、Ubuntu ランナーで十分。

### 却下: Xcode テスト (xcodebuild test)

iOS シミュレータ上のテスト実行は起動コストが高く、現テストは全て
`swift test` で実行可能なため不要。

## 影響

- CI パイプラインにテスト品質ゲートが追加される
- `build` と `test` が並列実行されるため、CI 全体の所要時間は増加しない
- 将来テストが増加した場合でも、この構成で対応可能
